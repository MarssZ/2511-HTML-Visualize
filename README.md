## 🎯 当前演示

### 1. 结构能：调整权重，显现结构

**核心概念**
通过调整权重阈值，让强连接自然浮现出清晰的几何结构（六角星）。

**适用场景**
向非技术人员演示"消除弱连接后，强连接自然显现结构"这一抽象概念。

**使用方法**
1. 打开 `demos/structure-energy/index.html`
2. 拖动滑块，观察混沌如何变为秩序
3. 当阈值为 1.0 时，只显示权重最高的边（六角星）

**技术特点**
- 原生 JavaScript（ES6+）
- HTML5 Canvas 渲染
- 零依赖，零构建
- 核心代码 < 100 行

**详细文档**
- [需求文档](specs/structure-energy/requirements.md)
- [设计文档](specs/structure-energy/design.md)

### 2. CAS 分形交互网络：三维认知空间

**核心概念**
通过2D分形交互网络 + 3D辅助视图，展示复杂适应系统的核心洞察：**任何一个代理都同时处于三种交互关系中**（系统内、跨系统、跨层级）。

**适用场景**
帮助管理者、产品经理、创业者理解：
- "同时性"的本质（不是三选一，而是三者并存）
- 分形嵌套结构（个人→团队→公司，每一层都有这三种模式）
- 扰动传播（任何变化都沿所有路径同时传播）

**使用方法**
1. 打开 `demos/cas-cube/index.html`
2. 观察金色核心代理（你）同时连接到三圈不同颜色的节点：
   - **紫色内圈**：6个同事（系统内互动）
   - **粉色中圈**：4个外部代理（跨系统互动）
   - **蓝色垂直轴**：上下级（跨层级互动）
3. **切换分形层级**：点击"个人/团队/公司"按钮，体验不同尺度的交互网络
4. **播放扰动传播**：点击按钮，观察三色波纹同时扩散
5. **悬停探索**：鼠标悬停在紫色/粉色/蓝色区域，查看详细说明

**核心交互**
- **分形层级切换**：个人 → 团队 → 公司（体验"层级中的层级"）
- **扰动传播演示**：三色波纹同时扩散（演示"同时性"）
- **维度高亮**：悬停某个维度，其他维度变半透明
- **3D辅助视图**：右上角旋转立方体，保留原文"立方体"隐喻

**技术特点**
- 原生 JavaScript（ES6+）
- 双Canvas渲染（主视图700x700 + 辅助视图200x200）
- 极坐标布局 + 3D投影算法
- 零依赖，零构建
- 核心代码 < 250 行

**详细文档**
- [需求文档](specs/cas-cube/requirements.md)
- [设计文档](specs/cas-cube/design.md)

## 🛠️ 技术栈

**极简原则：Less is More**

- **语言**：原生 JavaScript (ES6+)
- **渲染**：HTML5 Canvas API
- **样式**：内联 CSS
- **依赖**：零依赖（不引入Three.js、D3.js等任何库）
- **构建**：零构建（双击即运行）

---

## 💡 设计哲学

### 1. 极简主义
> "如果代码超过 200 行，说明方向错了。"

- 每个演示的核心逻辑 < 250 行（不含注释）
- 没有超过 3 层的嵌套
- 函数只做一件事并做好

### 2. 零破坏性
> "未来要加新功能，只需要添加字段，不改现有代码。"

- 数据结构设计为"只增不改"
- 演示之间完全独立，互不影响
- 向后兼容是铁律

### 3. 实用主义
> "解决真实问题，不是为论文服务。"

- 双击即运行，不需要教程
- 普通人 30 秒内能理解核心概念
- 拒绝过度设计（30行3D投影 vs 300KB的Three.js）

## 📖 如何添加新演示

### 步骤 1：创建演示目录
```bash
mkdir demos/your-demo-name
```

### 步骤 2：创建演示文件
在 `demos/your-demo-name/` 下创建：
- `index.html` - 演示页面（包含HTML/CSS/JS）
- `data.js` - 数据文件（可选，如果数据较多）

### 步骤 3：创建文档目录
```bash
mkdir specs/your-demo-name
```

### 步骤 4：编写文档
在 `specs/your-demo-name/` 下创建：
- `requirements.md` - 需求文档（用户故事、验收标准）
- `design.md` - 设计文档（数据结构、核心算法）

### 步骤 5：更新索引页
在根目录的 `index.html` 中添加新演示的卡片。

### 步骤 6：更新文档
更新本文件的"当前演示"部分。

详细流程参见 [CLAUDE.md](CLAUDE.md)。
