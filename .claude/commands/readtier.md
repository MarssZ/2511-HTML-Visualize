# 任务
在2分钟内，为内容分配处理层级（Tier 1/2/3或归档）。

**目标用户**：机制解构型学习者（追求底层原理，需要认知冲击和迁移价值）

**三级体系**：
- Tier 1（5-8分钟）：外部索引+概念挂载
- Tier 2（15-20分钟）：认知种子+触发器编码
- Tier 3（40-60分钟）：完整认知工具包

---

## 决策流程（120秒）

### 第一步：硬性排除（30秒）

触发归档：
- 纯工程实现，无方法论
- 与3个月研究方向无关
- 已掌握更优方案

---

### 第二步：前置检测（60秒）⭐ 核心

#### 母题识别（15秒）⭐ 新增

**问：这篇文章在解决哪个领域的哪个关键问题？**

**母题定位格式**：`[领域] - [母题]`
- 示例：`AI Agent - 工具调用优化`
- 示例：`计算机视觉 - 长尾分布问题`
- 示例：`NLP - 低资源语言建模`

**检验标准**：
- ✅ 能用一句话说清"这是在解决什么问题"
- ✅ 母题是该领域公认的重要问题（非自造词汇）
- ✅ 母题足够具体（不是"优化性能"，而是"优化什么的什么性能"）

**快速判断**：
- 如果15秒内识别不出明确母题 → 可能是低价值内容 → 考虑降级或归档
- 如果母题模糊不清 → 说明文章本身定位不清 → 降低Tier预期

---

#### 创新类型识别（25秒）

**问：核心贡献是什么？**

🔴 **范式创新**（理论/机制级）
- 打破了什么明确假设？[必须能具体说出]
- 例：Transformer打破"序列建模必须递归"

🟡 **方法创新**（算法/架构级）
- 有什么不可推导的巧妙设计？
- 例：LoRA低秩适配、CLIP对比学习

🟢 **工程创新**（整合/实践级）
- 整合了哪些已有技术？[列出公式]
- 例：ToolUniverse = RAG+网关+编排+监控

**判定规则**：
```
if 工程创新 && 用户=机制学习者:
    认知冲击 = 强制"低"
    迁移价值 = 最多"中"
    除非时效=高 → Tier 1或归档
```

---

#### 技术组合识别（20秒）

**如果是工程创新，拆解技术栈：**

```
每个组件对应什么现有技术？

示例（ToolUniverse）：
- 工具发现 → RAG检索
- 输入验证 → API网关（JSON Schema）
- 流程编排 → DAG引擎（Airflow）
- 质量监控 → CI/CD

生成类比："这就像用Spring Boot+Redis+Kafka构建电商系统"
→ 成熟技术的工程整合
→ 工程价值高，认知价值低
```

**可推导性测试**：有经验的人能独立设计吗？
- ✅ 能 → 认知价值=低
- ❌ 不能 → 需进一步分析

---

### 第三步：三维判别（45秒）

#### 维度一：🧠 认知冲击

**问：改变了我对底层机制的理解吗？**

- **高**：打破明确假设 / 解释长期困惑 / 新理论框架
- **中**：非常规技术方案 / 反直觉规律 / 不可推导的巧妙设计
  - ⚠️ **排除**：可推导的工程架构
- **低**：渐进优化 / 工程整合 / 最佳实践

**关键测试**：
1. 能具体说出打破了什么假设吗？→ 不能则≤中
2. 有经验的人能推导出来吗？→ 能则=低

---

#### 维度二：🔧 迁移价值

**问：这个洞察在其他场景的复用价值？**

**⚠️ 关键区分：认知洞察级 vs 工程实践级**

- **高（认知洞察级）**：
  - 底层机制、跨领域通用原理
  - 不可推导的创见（学习前后认知改变）
  - 例：Attention机制、Batch Normalization
  - 测试：去掉论文，我能独立想到吗？→ 不能

- **中（方法特定级）**：
  - 特定领域可迁移
  - 有创新但可类比推导

- **低（工程实践级）**：
  - 架构模式、设计模式（微服务、MVC）
  - 最佳实践组合
  - 测试：这是"常识"还是"洞察"？→ 常识

**关键测试**：
1. 是底层机制还是工程模式？→ 工程则≤中
2. 能通过类比推导吗？→ 能则=低
3. 学习后认知模型改变吗？→ 否则=低

---

#### 维度三：⏰ 时效要求

**问：不懂这个，3个月内会卡住吗？**

- **高**：核心研究方向、即将应用
- **中**：相关方向、可能用到
- **低**：外围领域、了解即可

---

### 决策矩阵

**Tier 3**：≥2个"高" 或 认知冲击=高+任一其他=高

**Tier 2**：1个"高"+≥1个"中" 或 ≥2个"中"且无"低"

**Tier 1**：其他情况

**归档**：硬性排除 或 工程创新且时效=低

---

### 3秒自问法（决策犹豫时）

1. **"打破了我什么假设?"**
   - 能说出 → Tier 3
   - 说不清 → 下一题

2. **"能用在其他3个地方吗?"**
   - 能（不可推导）→ Tier 2/3
   - 能（可推导）→ 工程级，降级
   - 不能 → 下一题

3. **"不懂会卡住吗?"**
   - 会 → 至少Tier 2
   - 不会 → Tier 1

---

## 关键约束

**⚠️ 警惕工程复杂度陷阱：**
- 系统庞大、组件众多 ≠ 认知深刻
- 工程完整性 ≠ 认知创新
- 整合600+工具 = 工程量大 ≠ 认知突破

**决策边界：**
- 不因"有趣"高估（有趣≠重要）
- 不因"复杂"高估（复杂≠深刻）
- 接受假阳性>假阴性（宁错升不错降）

```
【分级结果】：Tier X / 归档

【母题定位】：[领域] - [母题]

【创新类型】：🔴范式 / 🟡方法 / 🟢工程

【技术组合】：[如果工程创新]
  公式 = A + B + C
  类比："就像用X+Y+Z构建[场景]"
  可推导性：✅/❌

【三维评分】：
  🧠认知冲击 [高/中/低]
  🔧迁移价值 [高/中/低]
  ⏰时效要求 [高/中/低]

【核心理由】：[一句话]

【速判验证】：[3秒自问法验证]

【信心水平】：[高85%+ / 中70-85% / 低<70%]
```
