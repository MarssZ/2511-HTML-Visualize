# 内容类型参数对照表

## 文档定位

本文档是**三级认知流水线**（Tier 1/2/3）的配置手册，用于在统一框架下针对不同内容类型进行参数调整。

**核心原则**：框架统一，参数可调。

---

## 快速参考：四类内容的Tier分布建议

| 内容类型 | Tier 1比例 | Tier 2比例 | Tier 3比例 | 典型场景 |
|---------|-----------|-----------|-----------|---------|
| **学术论文** | 70% | 20% | 10% | 基线配置，适用于领域扫描 |
| **书籍** | 40% | 30% | 30% | 高价值密度，深度优先 |
| **文章/长文** | 75% | 20% | 5% | 快速筛选，精选深读 |
| **新闻/观点** | 85% | 15% | 0% | 信息索引为主 |

---

## 一、学术论文（基线配置）

### 特征分析
- **信息密度**：极高（方法论+实验+理论）
- **认知冲击潜力**：高（可能颠覆假设）
- **迁移价值**：中高（取决于领域相关性）
- **时效性**：中（前沿但非即时）

### Tier分布策略
- **Tier 1（70%）**：渐进式改进、特定场景方法、外围领域
- **Tier 2（20%）**：重要改进、跨领域方法、相关方向
- **Tier 3（10%）**：范式转变、核心方向、基础设施级

### 时间分配（10篇/天）
- Tier 1：7篇 × 8分钟 = 56分钟
- Tier 2：2篇 × 20分钟 = 40分钟
- Tier 3：1篇 × 60分钟 = 60分钟
- **总计**：156分钟（≈2.5小时）

### 工具调用组合
**Tier 1**：
- `/readtier1` → 快速索引生成
- 概念网络挂载（手动或 `/notelink` 轻量版）

**Tier 2**：
- `/readtier2` → 认知种子提取
- `/seed` → 种子四要素生成
- `/notelink` → 概念网络挂载

**Tier 3**：
- `/readtier3` → 完整工具包锻造
- `/schema` → 图式编码
- `/boom` → 认知炸弹
- `/notelink` → 深度概念挂载
- 可选：`/drawconcept` → 概念网络可视化

### 特殊注意
- **实验部分处理**：Tier 1只看主结论，Tier 2看消融实验，Tier 3理解为什么work
- **数学公式**：Tier 1/2不做推导，Tier 3理解直觉
- **代码实现**：不是必读项，除非是方法论核心

## 二、书籍（深度优先）

### 特征分析
- **信息密度**：中（有冗余，但体系完整）
- **认知冲击潜力**：极高（系统性颠覆）
- **迁移价值**：极高（跨领域框架）
- **时效性**：低（经典持久）
- **消费方式**：分阶段（检视→分析→主题）

### Tier分布策略
- **Tier 1（40%）**：工具书、入门读物、快速补盲
- **Tier 2（30%）**：重要但非核心领域、二次阅读
- **Tier 3（30%）**：认知基础设施、范式级著作

**关键差异**：书籍的Tier 3比例远高于论文，因为一本书的投资回报周期更长。

### 时间分配（分阶段消费）

**阶段一：决策前（5-10分钟）**
- `/spine-lite` → 灵魂判断（核心追问+关键转向+暗门密码）
- 判断：是否值得深读？

**阶段二：观其大略（Tier 1/2共用，20-30分钟）**
- `/spine-deep` → 结构分析（SPINE透视系统）
- `/mindmap` → 思维导图生成
- 产出：章节地图+阅读路线

**阶段三：探究明细（Tier 2/3分化）**
- **Tier 2路径**：
  - 选择3-5个关键章节精读
  - `/logline` → 认知路径规划（按章节）
  - `/key` → 启明之钥（引导式阅读）
  - `/seed` → 认知种子提取
  - 总时间：60-90分钟

- **Tier 3路径**：
  - 全书精读（按计划分批）
  - 每章使用 `/logline` + `/key`
  - 每Part使用 `/cognitive` → 认知画布
  - 全书使用 `/story_spine` → 故事骨架
  - 总时间：6-10小时（分多日）

**阶段四：满载而归（所有Tier）**
- `/do` → 实践起点（认知扳机+三维锚点）
- `/drawconcept` → 概念网络图
- `/notelink` → 深度挂载

### 工具调用组合（完整流程）

```
[决策] /spine-lite
    ↓
[值得读？]
    ↓ Yes
[观其大略] /spine-deep + /mindmap
    ↓
[Tier判断]
    ↓
[Tier 1] → 只保留思维导图 + 核心问题 + /do
    ↓
[Tier 2] → 选3-5章 → /logline + /key + /seed + /do
    ↓
[Tier 3] → 全书精读 → /logline + /key + /cognitive + /story_spine + /do + /drawconcept
```

### 特殊注意
- **不要一次性读完**：书籍应分阶段消费，每阶段决策是否继续
- **检视阅读的价值**：Tier 1的书，检视阅读就足够，不要有负罪感
- **主题阅读优先**：如果是系统学习某领域，考虑多本书的主题阅读
- **利用目录结构**：书籍的章节标题是最好的价值信号

## 三、文章/长文（快速筛选）

### 特征分析
- **信息密度**：中（介于论文和书籍之间）
- **认知冲击潜力**：中（单点洞察）
- **迁移价值**：中（局部方法）
- **时效性**：中高（热点话题）
- **篇幅**：3000-10000字

### Tier分布策略
- **Tier 1（75%）**：泛读文章、信息速览、观点扫描
- **Tier 2（20%）**：深度长文、系统方法论、高质量分析
- **Tier 3（5%）**：经典长文、体系性文章（罕见）

### 时间分配（20篇/天）
- Tier 1：15篇 × 5分钟 = 75分钟
- Tier 2：4篇 × 15分钟 = 60分钟
- Tier 3：1篇 × 30分钟 = 30分钟
- **总计**：165分钟（≈2.5小时）

### 工具调用组合
**Tier 1**（最简模式）：
- 快速扫描标题、小标题、首尾段
- 手动记录：1句话核心观点 + 1个触发场景
- 可选：挂载到相关概念页面

**Tier 2**：
- `/seed` → 认知种子（根本问题+奇异答案+迁移地图）
- `/boom_simple` → 认知炸弹单一版（三重觉醒引力场）
- `/notelink` → 概念挂载

**Tier 3**（极少使用）：
- `/cognitive` → 认知画布（完整分析）
- `/drawconcept` → 概念网络图
- `/notelink` → 深度挂载

### 特殊注意
- **小标题是金矿**：文章的小标题直接揭示价值密度
- **警惕标题党**：标题吸引但内容空洞的文章应快速放弃
- **作者权威**：同一作者的高质量文章可批量提升到Tier 2
- **时效衰减**：新闻类文章的价值半衰期极短，优先Tier 1

## 四、新闻/观点（索引为主）

### 特征分析
- **信息密度**：低（单一事实或观点）
- **认知冲击潜力**：低（罕见颠覆）
- **迁移价值**：低（情境特定）
- **时效性**：极高（即时消费）
- **篇幅**：500-2000字

### Tier分布策略
- **Tier 1（85%）**：日常新闻、热点观点、行业动态
- **Tier 2（15%）**：深度调查、系统分析、独特视角
- **Tier 3（0%）**：不建议投入（除非极特殊）

### 时间分配（30条/天）
- Tier 1：25条 × 2分钟 = 50分钟
- Tier 2：5条 × 10分钟 = 50分钟
- **总计**：100分钟（≈1.5小时）

### 工具调用组合
**Tier 1**（闪电模式）：
- 阅读标题 + 首段
- 判断：事实(fact) or 观点(opinion) or 噪音(noise)
- 记录格式：
  - 事实：`[日期] 发生了什么 + 影响范围`
  - 观点：`[作者] 对X的观点 + 1句话论据`
  - 噪音：直接跳过

**Tier 2**：
- 完整阅读
- `/seed` → 认知种子（轻量版）
  - 根本问题：这篇报道/分析要解决什么困惑
  - 奇异答案：核心洞察是什么
  - 萌发条件：什么场景会想起
- 挂载到相关概念（手动链接）

### 特殊注意
- **建立过滤规则**：同类新闻只看第一条，后续跳过
- **观点vs洞察**：观点≠洞察，只有带证据链的观点才值得Tier 2
- **事实索引**：重大事实应记录到时间线，而非概念网络
- **警惕回声室**：同一观点的不同表述不要重复处理

## 五、跨类型通用原则

### 价值信号识别
无论哪种类型，以下信号提示应提升Tier：
1. **认知失调点**："这不对啊...为什么？"的时刻
2. **结构同构**："这和XX问题本质一样！"的联想
3. **黑话破解**："原来这个概念是这个意思！"的顿悟
4. **框架颠覆**："我一直理解错了！"的震撼
5. **可迁移工具**："这个方法能用在3+个地方"的预感

### 降级信号识别
以下信号提示应降低Tier或归档：
1. **信息冗余**："这个我已经知道了"
2. **假深刻**："看起来很厉害但说不出所以然"
3. **远离需求**："有趣但3个月内用不到"
4. **质量低下**："逻辑混乱/证据不足"
5. **纯工程实现**："没有方法论，只有代码"

### 时间管理策略
**每日能量分配**：
- 上午（能量充沛）：Tier 3 + Tier 2
- 下午（能量中等）：Tier 2 + Tier 1
- 晚上（能量较低）：Tier 1批量处理

**批量处理技巧**：
- 同类型内容连续处理（减少切换成本）
- Tier 1可批量5-10篇后统一挂载概念网络
- Tier 3一天最多1篇（需要认知沉淀）

### 质量复盘机制
**每周复盘（15分钟）**：
- 检查Tier分布是否健康（避免全是Tier 3或全是Tier 1）
- 回溯验证：上周的Tier 3是否值得1小时？
- 误判纠正：Tier 1中频繁回想的内容 → 应该是Tier 2
- 调整策略：根据实际情况微调判别阈值

## 六、特殊场景处理

### 场景1：主题阅读
当系统学习某主题时（如"强化学习"），需要阅读5-10本书/论文：

**策略**：
1. 第1本：用Tier 3深度处理（建立框架）
2. 第2-3本：用Tier 2补充（填补空白）
3. 第4-10本：用Tier 1扫描（确认无新增）

**工具链**：
- 先用 `/findbook` 找到该领域的"圣经"
- 第1本用完整Tier 3流程
- 后续书籍用 `/spine-lite` 快速判断差异点
- 用 `/crossover` 寻找跨界连接

### 场景2：紧急学习
当需要快速掌握新领域（如3天内学会某框架）：

**策略**：
1. 忽略Tier分级，直接选3-5份核心材料
2. 全部用Tier 2处理（不求完美，先建框架）
3. 边学边用，在实践中迭代认知

**工具链**：
- `/seed` → 快速建立认知种子
- `/do` → 立即找到实践起点
- 跳过概念网络挂载（事后补）

### 场景3：信息爆炸日
当某天收到30+篇必读论文：

**策略**：
1. 快速分类：核心(5篇) + 相关(10篇) + 外围(15篇)
2. 核心：正常Tier分级
3. 相关：全部降级处理（Tier 2→Tier 1）
4. 外围：只看标题+摘要，记录到"待处理"列表

**工具链**：
- `/readtier` → 2分钟快速分级
- 批量Tier 1处理（每篇5分钟）
- 核心论文延后到第二天处理

## 七、个性化调整指南

### 如何校准你的Tier阈值？

**信号1：经常后悔**
- 症状：Tier 1的内容频繁回想，觉得应该深读
- 调整：放宽Tier 2的准入标准（降低"认知冲击"阈值）

**信号2：时间超支**
- 症状：每天总是超过预算时间
- 调整：
  - 提高Tier 3的准入标准（只留10%真正颠覆的）
  - 或缩短单篇时间（Tier 2从20min→15min）

**信号3：记不住**
- 症状：Tier 2的内容很快遗忘
- 调整：
  - 加强记忆锚点设计（增加震撼数字/隐喻）
  - 或提高触发器设计质量

**信号4：用不上**
- 症状：学了很多但实践中想不起来
- 调整：
  - 强化"萌发条件"设计（更具体的场景）
  - 或增加实践环节（每个Tier 2都要有 `/do`）

### 建立你的个人预设

根据你的研究领域和学习风格，建立个性化预设：

**示例：计算机视觉研究者**
```
论文类型预设：
- 检测/分割方法：默认Tier 1（除非SOTA突破）
- Transformer变体：默认Tier 2（架构创新）
- 自监督学习：默认Tier 2（方法论价值高）
- 理论分析：默认Tier 3（深度理解）

作者信任列表：
- Kaiming He / Yann LeCun / ... → 默认提升1个Tier
```

## 使用建议

1. **从论文基线开始**：先用论文配置运行2周，熟悉流程
2. **逐步扩展**：再引入书籍和文章的处理
3. **建立仪式感**：每天固定时间处理Tier 1（批量），Tier 3（专注）
4. **定期校准**：每周复盘，每月调整参数
5. **工具熟练度**：先掌握核心工具（/seed, /notelink, /do），再扩展

**记住**：这是一个成长系统，会随着你的使用而进化。不要追求一开始就完美，而是在实践中持续优化。

---

**最后的元认知提醒**：

这套系统的目标不是"处理更多信息"，而是"建立更强认知"。

如果发现自己陷入"完成Tier数量"的焦虑，停下来，问自己：

> "我的认知模型在最近一周发生了什么真正的改变？"

如果答案是"没有"，说明你在用战术的勤奋掩盖战略的懒惰。

**少即是多。深度优于广度。**
