# 工具整合手册

## 文档定位

本手册是**三级认知流水线**与你现有认知工具库的整合指南。它将告诉你：
- 每个Tier应该调用哪些工具
- 工具之间的调用顺序和协作关系
- 不同内容类型的工具组合策略
- 实战案例和最佳实践

**核心原则**：工具为认知服务，而非认知为工具服务。

## 二、Tier 1 工具链（指针层：5-8分钟）

### 2.1 核心目标
创建外部索引，不追求深度理解。

### 2.2 标准流程

```
[输入] 论文PDF/URL
    ↓
[主工具] /readtier1 (5-8分钟)
    ↓
    产出：
    - 📌 核心贡献（1句话）
    - 🔑 关键信息（领域、方法类型、创新点、适用场景）
    - 🎯 触发场景（2个）
    - 📊 概念网络挂载（手动）
    ↓
[可选增强]
    - 如果有明显的概念连接 → 手动链接到概念页面
    - 如果是系列工作 → 链接到相关论文
    ↓
[输出] 论文索引笔记 + 概念网络节点
```

### 2.3 工具组合策略

**最简模式（5分钟）**：
```
/readtier1 → 生成索引 → 完成
```

**标准模式（8分钟）**：
```
/readtier1 → 生成索引 → 手动概念挂载 → 完成
```

**不推荐使用的工具**：
- ❌ `/seed`（时间超支，Tier 1不需要）
- ❌ `/boom`（过度分析）
- ❌ `/cognitive`（完全超出范围）

### 2.4 质量检查

**3秒自测**：
1. "6个月后，我能通过这个索引想起这篇论文吗？" → 如果不能，补充1句描述
2. "如果搜索某个概念，能找到这篇论文吗？" → 如果不能，补充标签
3. "概念网络挂载是否有效？" → 至少1个主概念

## 三、Tier 2 工具链（种子层：15-20分钟）

### 3.1 核心目标
提取认知种子，设计触发器，确保可被唤醒。

### 3.2 标准流程（论文）

```
[输入] 论文PDF/URL
    ↓
[Step 1] 精准精读（7分钟）
    - 手动：Abstract + Intro + 方法核心 + 实验关键图表
    ↓
[Step 2] 认知种子提取（5分钟）
    [主工具] /seed
    ↓
    产出：
    - 🌱 根本问题
    - 💡 奇异答案
    - 🔑 记忆锚点（震撼数字+核心案例+同构比喻）
    - 🗺️ 迁移地图（3个场景）
    - ⚠️ 边界条件
    ↓
[Step 3] 认知冲击识别（3分钟）
    [可选工具] /boom_simple
    ↓
    产出：
    - 颠覆引力场：打破了什么假设
    - 痛苦共振场：解决了谁的痛苦
    - 行动催化场：如何立即验证
    ↓
[Step 4] 概念网络挂载（3分钟）
    [主工具] /notelink
    ↓
    产出：
    - 上级概念（理论基础）
    - 下级概念（具体应用）
    - 同级概念（平行方案）
    - 跨界连接（结构同构）
    ↓
[Step 5] 质量检查（2分钟）
    - 唤醒测试：6个月后能想起吗？
    - 记忆测试：有震撼锚点吗？
    - 迁移测试：能说出3个应用场景吗？
    ↓
[输出] 认知种子笔记 + 触发器 + 概念网络
```

### 3.3 工具组合策略

**最简模式（15分钟）**：
```
精读 → /seed → 手动概念挂载 → 完成
```

**标准模式（20分钟）**：
```
精读 → /seed → /boom_simple → /notelink → 完成
```

**加强模式（25分钟，超时但值得）**：
```
精读 → /seed → /boom_simple → /notelink → /do（实践起点）→ 完成
```

### 3.4 工具替代方案

如果时间不足，可按优先级调整：

**优先级排序**：
1. 🔴 必选：`/seed`（核心种子）
2. 🟡 推荐：`/notelink`（概念挂载）
3. 🟢 可选：`/boom_simple`（认知冲击）
4. 🟢 可选：`/do`（实践转化）

**时间压缩策略**：
- 20分钟 → 15分钟：跳过 `/boom_simple`
- 15分钟 → 12分钟：手动概念挂载代替 `/notelink`

### 3.5 质量检查工具

使用 `/notecheck` 进行事后质量检查（可选，3分钟）：
- 信噪比优化
- 结构化程度
- 可复用性

## 四、Tier 3 工具链（锻造层：40-60分钟）

### 4.1 核心目标
构建完整认知工具包：图式+透镜+对撞机。

### 4.2 标准流程（论文）

```
[输入] 论文PDF/URL
    ↓
[Step 1] 全文精读+结构化笔记（20分钟）
    - 手动：三遍阅读法
      - 第一遍：结构扫描（5分钟）
      - 第二遍：深度理解（10分钟）
      - 第三遍：批判阅读（5分钟）
    ↓
[Step 2] 图式编码（8分钟）
    [推荐工具] 手动构建 或 调用现有schema工具
    ↓
    产出：
    - 概念关系图（Mermaid）
    - 精准隐喻（结构同构）
    - 核心公式的直觉
    ↓
[Step 3] 认知透镜锻造（15分钟）
    [工具组合] /cognitive + /drill（可选）
    ↓
    产出（/cognitive）：
    - 🌟 星系核心（认知模型）
    - ⭐ 恒星（关键概念）
    - 🌌 星座（关键模式）
    - 🕳️ 虫洞（核心隐喻）
    - 🪐 行星系统（关键案例）
    - 💥 超新星（认知失调点）
    - 🛸 登陆舱（可行动工具）
    ↓
    可选增强（/drill）：
    - 第一性原理拆解
    - 概念边界勘测
    ↓
[Step 4] 思想对撞机（10分钟）
    [工具组合] /crossover + /concept
    ↓
    产出（/crossover）：
    - 核心模式抽象
    - 跨界同构发现（5+个领域）
    - 创新火花
    ↓
    可选增强（/concept, /archaeologist, /genealogist）：
    - 概念溯源
    - 概念起源
    - 概念系谱
    ↓
[Step 5] 概念网络深度挂载（5分钟）
    [主工具] /notelink + /drawconcept
    ↓
    产出（/notelink）：
    - 四维连接（叙词+母题+层级+相关）
    - 创建概念页面（如不存在）
    ↓
    产出（/drawconcept）：
    - 生命之树形态知识图谱
    - 可视化概念网络
    ↓
[Step 6] 质量检查+内化测试（5分钟）
    [工具] /check（可选）
    ↓
    - 费曼测试：能讲给外行听吗？
    - 迁移测试：能说出5+个应用吗？
    - 边界测试：能预测失效场景吗？
    ↓
[Step 7] 实践转化（加时，可选5分钟）
    [主工具] /do
    ↓
    产出：
    - 核心认知扳机
    - 三维锚点（问题+场景+概念）
    - 最小启动单元
    ↓
[输出] 完整认知工具包 + 可视化图谱 + 实践指南
```

### 4.3 工具组合策略

**最简模式（40分钟）**：
```
精读(20) → 图式编码(8) → /cognitive(12) → /notelink(5) → 质量检查(5)
```

**标准模式（50分钟）**：
```
精读(20) → 图式编码(8) → /cognitive(12) → /crossover(8)
→ /notelink(3) → /drawconcept(5) → 质量检查(5)
```

**完整模式（60分钟）**：
```
精读(20) → 图式编码(8) → /cognitive(10) → /drill(5)
→ /crossover(8) → /concept(3) → /notelink(3) → /drawconcept(5)
→ 质量检查(3) → /do(5)
```

**深度研究模式（90分钟，超标但适用于核心论文）**：
```
精读(30) → 图式编码(10) → /cognitive(15) → /drill(10)
→ /crossover(10) → /concept+/archaeologist+/genealogist(10)
→ /notelink(5) → /drawconcept(10) → /do(5) → /practice(5)
→ 质量检查(5) → /notecheck(5)
```

### 4.4 工具优先级排序

**必选（核心三件套）**：
1. 🔴 `/cognitive` - 认知画布（核心分析）
2. 🔴 `/notelink` - 概念网络挂载
3. 🔴 质量检查（费曼+迁移+边界）

**强烈推荐**：
4. 🟡 `/crossover` - 跨界发现（迁移价值）
5. 🟡 `/drawconcept` - 可视化（记忆强化）
6. 🟡 `/do` - 实践转化（知行合一）

**可选增强**：
7. 🟢 `/drill` - 第一原理拆解（概念深度）
8. 🟢 `/concept` - 概念溯源（历史脉络）
9. 🟢 `/check` - 自我审视（元认知）
10. 🟢 `/practice` - 刻意练习设计

### 4.5 时间压缩策略

当时间不足60分钟时：

**60分钟 → 50分钟**：
- 跳过 `/drill` 和 `/concept`

**50分钟 → 40分钟**：
- 跳过 `/drawconcept`
- `/crossover` 压缩到5分钟（只找3个跨界）

**40分钟 → 30分钟**（极限压缩，不推荐）：
- 精读压缩到15分钟
- 只用 `/cognitive` + `/notelink`
- 跳过所有可选工具

## 五、书籍专用工具链

### 5.1 分阶段消费策略

书籍不同于论文，需要分阶段决策。

#### 阶段一：决策前（5-10分钟）

```
[输入] 书籍信息（书名/封面/目录）
    ↓
[主工具] /spine-lite
    ↓
产出：
    - 🎯 核心追问
    - ⚡ 关键转向
    - 💡 一念之转（认知裂变）
    - 🔑 暗门密码（核心洞见）
    ↓
[决策] 是否值得深读？
    ↓ Yes
    进入阶段二
    ↓ No
    归档或降级为Tier 1
```

#### 阶段二：观其大略（20-30分钟）

```
[输入] 书籍全文
    ↓
[主工具] /spine-deep + /mindmap
    ↓
产出（/spine-deep）：
    - Survey（扫描）
    - Preview（预览）
    - Index（挖掘）
    - Navigate（导航）
    - Evaluate（评估）
    ↓
产出（/mindmap）：
    - 完整章节结构图
    - 核心内容节点（带原文锚点）
    - 配色语义化
    ↓
[决策] Tier判断
    ↓
    Tier 1 → 只保留mindmap → 完成
    ↓
    Tier 2 → 选择3-5个关键章节 → 进入阶段三
    ↓
    Tier 3 → 全书精读计划 → 进入阶段三
```

#### 阶段三：探究明细（Tier 2：60-90分钟 / Tier 3：6-10小时）

**Tier 2路径**：
```
[输入] 选定的3-5个关键章节
    ↓
对每个章节：
    [工具] /logline
    ↓
    产出：
    - 认知路径地图（1-3条路线）
    - 每个节点的Logline（承上启下）
    - 叙事骨架（解谜式/工具建造式/世界观颠覆式）
    ↓
    [工具] /key（启明之钥）
    ↓
    产出：
    - 引导式阅读体验
    - 一步步理解核心内容
    ↓
精读完成后：
    [工具] /seed
    ↓
    产出：认知种子（每章1个）
    ↓
最终：
    [工具] /do
    ↓
    产出：实践起点
```

**Tier 3路径**：
```
[输入] 全书分批精读
    ↓
对每一章：
    [工具组合] /logline + /key（同Tier 2）
    ↓
对每一Part：
    [工具] /cognitive
    ↓
    产出：认知画布分析
    ↓
全书完成后：
    [工具] /story_spine
    ↓
    产出（基于 /spine-lite + /spine-deep 的综合）：
    - 🎬 认知困境
    - 😰 旧范式的失效
    - 💡 视角革命
    - 🧐 核心主张
    - 🛠️ 论证体系
    - 🏆 思想禀赋
    - 😈 固有局限
    - 🌍 使用指南
    ↓
    [工具] /do + /practice
    ↓
    产出：完整实践方案
```

#### 阶段四：满载而归（所有Tier通用）

```
[工具组合] /drawconcept + /notelink + /restructure（可选）
    ↓
产出：
    - 概念网络可视化图谱
    - 四维概念挂载
    - （可选）概念网络重构
```

### 5.2 书籍工具优先级

**Tier 1书籍**：
1. 🔴 `/spine-lite`（必选）
2. 🟡 `/mindmap`（推荐）
3. 🟢 `/do`（可选，如果有实践价值）

**Tier 2书籍**：
1. 🔴 `/spine-lite` + `/spine-deep` + `/mindmap`
2. 🔴 `/logline` + `/key`（对选定章节）
3. 🔴 `/seed`（每章）
4. 🟡 `/do`
5. 🟢 `/drawconcept` + `/notelink`

**Tier 3书籍**：
1. 🔴 全套（/spine-lite → /spine-deep → /mindmap → /logline → /key → /cognitive → /story_spine）
2. 🔴 `/do` + `/practice`
3. 🟡 `/drawconcept` + `/notelink`
4. 🟢 `/restructure`（如需深度整合）

## 六、跨Tier通用工具

### 6.1 概念网络挂载工具

**`/notelink` - 知识关联编织师**

适用场景：所有Tier，但深度不同

**Tier 1用法（2分钟）**：
- 只做叙词标注（统一术语）
- 手动链接1-2个主要概念

**Tier 2用法（3分钟）**：
- 叙词+母题
- 上下级关系（理论基础+具体应用）
- 同级关系（平行方案）

**Tier 3用法（5分钟）**：
- 完整四维连接（叙词+母题+层级+相关）
- 创建新概念页面
- 跨界连接（结构同构）

---

### 6.2 实践转化工具

**`/do` - 实践工匠**

```
2. [7-12分钟] /seed 认知种子提取
   输入：论文PDF
   产出：
   - 🌱 根本问题：标准Attention在长序列上泛化能力差
   - 💡 奇异答案：用相对位置而非绝对位置编码
   - 🔑 记忆锚点：
     - 震撼数字：15%提升，30%参数减少
     - 核心案例：机器翻译任务，序列长度从512泛化到2048
     - 同构比喻：就像用"相对坐标"而非"绝对坐标"导航
   - 🗺️ 迁移地图：
     1. 图像的位置编码（Vision Transformer）
     2. 时间序列预测（长期依赖）
     3. 推荐系统（序列建模）

3. [12-15分钟] /notelink 概念网络挂载
   输入：上述认知种子
   产出：
   - 上级概念：[[Concept - Attention机制]]

4. [15-18分钟] 质量检查

   - ✅ 记忆测试：有"相对坐标导航"隐喻

   - ✅ 简洁测试：种子<500字

总结：18分钟，完成Tier 2标准流程，产出可被唤醒的认知种子。
```

## 八、常见问题

### Q1：工具太多，记不住怎么办？

**A**：建立工具使用习惯的三步法：

**第一周：只用核心三件套**
- Tier 1：`/readtier1`
- Tier 2：`/seed` + `/notelink`
- Tier 3：`/cognitive` + `/notelink`

**第二周：加入可视化**
- 所有Tier：+ `/drawconcept`（一周用1次）

**第三周：加入实践转化**
- 所有Tier：+ `/do`

**第四周及以后：按需扩展**
- 根据实际需求，逐步引入其他工具

---

### Q2：时间总是超支怎么办？

**A**：三个压缩策略：

**策略1：严格时间盒**
- 设置计时器，时间到立即停止
- 宁可降低质量，不要超时（系统稳定性>单次完美性）

**策略2：工具优先级**
- 参考本文档的"工具优先级排序"
- 时间不足时，按优先级删减

**策略3：降级处理**
- Tier 3超时 → 降为Tier 2
- Tier 2超时 → 降为Tier 1
- 不要有负罪感，这是系统设计的一部分

---

### Q3：如何判断一个工具是否值得用？

**A**：三个判断标准：

**标准1：边际收益**
- 这个工具带来的认知增量 > 它消耗的时间成本？
- 例如：`/seed`（5分钟，高增量）vs `/genealogist`（10分钟，低增量）

**标准2：复用性**
- 这个工具的产出能被未来复用吗？
- 例如：`/drawconcept`（产出可分享）vs 手动笔记（难复用）

**标准3：不可替代性**
- 这个工具提供的功能，手动做很难达到吗？
- 例如：`/notelink`（自动四维连接）vs 手动概念挂载（可替代）

---

### Q4：书籍和论文能用同一套工具吗？

**A**：框架统一，工具有别。

**统一框架**：
- 都遵循三级流水线（Tier 1/2/3）
- 都需要概念网络挂载（`/notelink`）
- 都需要实践转化（`/do`）

**工具差异**：
- 书籍阅读前多一步：`/spine-lite`（决策）
- 书籍需要分阶段：`/spine-deep` → `/mindmap` → `/logline` → `/key`
- 论文更强调方法论：`/seed` → `/cognitive` → `/crossover`

**参考**：本文档"五、书籍专用工具链"

---

### Q5：概念网络越来越乱怎么办？

**A**：定期维护+重构

**每周维护（10分钟）**：
- 检查是否有孤立节点（没有链接的笔记）
- 合并重复概念
- 补充缺失链接

**每月重构（30分钟）**：
- 使用 `/restructure` 重新梳理某个主题的概念网络
- 删除过时或价值低的节点
- 创建新的概念索引页

**可视化检查**：
- 定期用 `/drawconcept` 绘制局部概念网络
- 如果图谱混乱，说明需要重构

---

### Q6：实践转化工具（/do, /practice）什么时候用？

**A**：根据内容性质判断

**优先使用场景**：
- 方法论类内容（如《刻意练习》）
- 实用技能类论文（如"提示词工程"）
- 实践指南类文章

**不需要使用场景**：
- 纯理论论文（如数学证明）
- 调查报告类文章（只需知道事实）
- 观点评论类内容

**使用时机**：
- Tier 2/3：完成认知种子提取后
- 在概念挂载之前（实践锚点有助于记忆）

### Q8：工具调用失败或产出质量低怎么办？

**A**：四步诊断法

**步骤1：检查输入质量**
- 提示词是否完整？
- 输入的内容是否足够（如PDF是否完整）？

**步骤2：调整工具参数**
- 是否需要提供更多上下文？
- 是否需要分步调用（如先`/seed`再`/notelink`）？

**步骤3：降级或替代**
- 该工具是否必需？能否手动完成？
- 能否用更简单的工具替代（如用`/boom_simple`代替`/boom`）？

**步骤4：反馈迭代**
- 记录失败案例
- 调整个人工具使用策略
- 可能某些工具不适合你的认知风格

---

### Q9：如何避免工具成为形式主义？

**A**：时刻问自己三个问题

**问题1：认知模型改变了吗？**
- 不是"我完成了X个工具调用"
- 而是"我的思维模型在这周发生了什么改变"

**问题2：能立即复述吗？**
- 合上笔记，能用自己的话讲出核心内容吗？
- 如果不能，说明工具没有服务于认知内化

**问题3：有行动冲动吗？**
- 读完后有"我要试试"的感觉吗？
- 如果没有，可能是内容不适合你，或工具链设计有问题

**核心原则**：
> 工具是认知的脚手架，不是认知的替代品。
> 如果工具让你离"真实理解"更远，立即停用。

## 十、最后的元认知提醒

这份手册提供了40+个工具的整合方案，但请记住：

> **工具的价值不在于使用了多少，而在于解决了多少认知问题。**

**三个避免陷阱**：

1. **完美主义陷阱**：不要追求"每篇论文都用完整工具链"
   - 70%的内容用最简流程就够了
   - 只有10%的内容值得倾注全部火力

2. **工具收集陷阱**：不要成为"工具收藏家"
   - 掌握5个核心工具 > 了解50个工具
   - 熟练使用 > 全面覆盖

3. **形式主义陷阱**：不要让工具成为"打卡任务"
   - 问自己："这个工具让我的认知发生改变了吗？"
   - 如果答案是"没有"，停用它

**一个核心原则**：

> **少即是多。深度优于广度。**
> **让工具服务于认知进化，而非相反。**
